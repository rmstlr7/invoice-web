# Invoice Web MVP PRD

## 🎯 핵심 정보

**목적**: Notion에 저장된 인보이스를 클라이언트가 안전하게 웹으로 확인하고 PDF로 다운로드할 수 있는 시스템 제공
**사용자**: 견적서/인보이스 발급자(판매자)와 이를 받는 고객(클라이언트)

---

## 🚶 사용자 여정

```
1. 발급자: 인보이스 정보를 Notion Database에 입력
   ↓ [고유 URL 생성/공유]

2. 클라이언트: 공유받은 URL 접속
   ↓

3. 인보이스 상세 페이지
   ├─ 고객명, 금액, 발급일, 만기일 확인
   ├─ 인보이스 상태(미납/납부완료/연체/취소) 확인
   ↓ [PDF 다운로드 버튼 클릭]

4. PDF 생성 & 다운로드
   ↓

5. 완료 → 추가 액션
   ├─ 인보이스 보관
   └─ 판매자에게 연락 (외부)
```

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID       | 기능명             | 설명                                           | MVP 필수 이유                     | 관련 페이지          |
| -------- | ------------------ | ---------------------------------------------- | --------------------------------- | -------------------- |
| **F001** | 인보이스 상세 조회 | Notion DB에서 인보이스 정보 조회 및 표시       | 서비스의 핵심 가치 제공           | 인보이스 상세 페이지 |
| **F002** | 인보이스 상태 표시 | 미납/납부완료/연체/취소 상태를 시각적으로 표시 | 클라이언트의 결제 상태 파악 필수  | 인보이스 상세 페이지 |
| **F003** | PDF 다운로드       | 인보이스를 PDF 파일로 다운로드                 | 인보이스의 영구 보관 및 인쇄 필요 | 인보이스 상세 페이지 |

### 2. MVP 필수 지원 기능

| ID       | 기능명        | 설명                                            | MVP 필수 이유            | 관련 페이지          |
| -------- | ------------- | ----------------------------------------------- | ------------------------ | -------------------- |
| **F010** | 고유 URL 접속 | 공유받은 고유 URL로 인보이스 페이지 직접 접근   | 인증 없이 빠른 접근 가능 | 인보이스 상세 페이지 |
| **F011** | 에러 처리     | 존재하지 않는 인보이스 접근 시 안내 메시지 표시 | 사용자 경험 기본         | 에러 페이지          |

### 3. MVP 이후 기능 (제외)

- 인보이스 목록 페이지
- 인보이스 생성/수정/삭제 (발급자용 관리 페이지)
- 사용자 계정 및 인증 시스템
- 결제 연동 (결제 게이트웨이)
- 메일 알림 기능
- 영수증 발급

## 📄 페이지별 상세 기능

### 인보이스 상세 페이지

> **구현 기능:** `F001`, `F002`, `F003` | **진입:** 고유 URL 접속

| 항목            | 내용                                                                                                                                                                                                                                                                                                   |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **역할**        | 인보이스의 모든 정보를 한눈에 표시하고 PDF 다운로드 지원하는 핵심 페이지                                                                                                                                                                                                                               |
| **진입 경로**   | 발급자가 공유한 고유 링크 직접 접속 (고유 ID를 URL에 포함)                                                                                                                                                                                                                                             |
| **사용자 행동** | 클라이언트가 페이지 접속 → 인보이스 내용 확인 → PDF 다운로드 버튼 클릭                                                                                                                                                                                                                                 |
| **주요 기능**   | • 고객명 표시<br>• 인보이스 번호 표시<br>• 청구 금액 표시 (한국 원화 KRW)<br>• 발급일 표시 (한국식 날짜 형식)<br>• 만기일 표시 (한국식 날짜 형식)<br>• 인보이스 상태 표시 (상태별 색상 구분: 납부완료=초록색, 미납=노란색, 연체=빨간색, 취소=회색)<br>• 설명/비고 필드 표시<br>• **PDF 다운로드** 버튼 |
| **다음 이동**   | PDF 다운로드 완료 후 페이지 유지, 다른 액션 없음                                                                                                                                                                                                                                                       |

---

### 에러 페이지

> **구현 기능:** `F011` | **조건:** 존재하지 않는 ID 접속 시

| 항목            | 내용                                                                                                            |
| --------------- | --------------------------------------------------------------------------------------------------------------- |
| **역할**        | 잘못된 인보이스 ID로 접속했을 때 사용자에게 알려주는 페이지                                                     |
| **진입 경로**   | F010 고유 URL 접속 → 존재하지 않는 ID 감지 → 자동 리디렉션                                                      |
| **사용자 행동** | 에러 메시지 확인 후 판매자에게 연락 (외부)                                                                      |
| **주요 기능**   | • 에러 메시지 명확 표시 (예: "인보이스를 찾을 수 없습니다")<br>• 발생 원인 설명<br>• 판매자에게 문의하도록 유도 |
| **다음 이동**   | 에러 페이지 유지 또는 뒤로가기                                                                                  |

## 🗄️ 데이터 모델

### Invoice (인보이스)

Notion Database에서 관리되는 주요 필드

| 필드          | 설명                                    | 타입          |
| ------------- | --------------------------------------- | ------------- |
| id            | Notion 페이지 ID (고유 식별자)          | UUID (Notion) |
| invoiceNumber | 인보이스 번호                           | Text          |
| clientName    | 고객명                                  | Text          |
| amount        | 청구 금액                               | Number        |
| issueDate     | 발급일                                  | Date          |
| dueDate       | 만기일                                  | Date          |
| status        | 인보이스 상태 (미납/납부완료/연체/취소) | Select        |
| description   | 설명/비고                               | Text (Long)   |
